Part 1 · ≈80% of the guide

Free manual approach: seeing & detecting CAPTCHA

This part is deliberately low-level. The goal is to trigger Amazon's bot protection, learn to recognize a CAPTCHA response, and understand why simple header tweaks rarely "solve" CAPTCHA at scale.

1. Minimal request that likely triggers CAPTCHA

amazon_captcha_minimal.pypython
import requests

URL = "https://www.amazon.com/dp/B096N2MV3H"

response = requests.get(URL)
print("Status code:", response.status_code)
print(response.text[:500])
runbash
python examples/amazon_captcha_minimal.py

Typical outcomes: status 503, 403, or even 200 with a bot-detection / CAPTCHA page instead of real product HTML. Hints include references to "robot check", text asking you to "type the characters you see", or URLs containing captcha.

2. Saving the HTML and inspecting the CAPTCHA page

amazon_captcha_save.pypython
import requests

URL = "https://www.amazon.com/dp/B096N2MV3H"
response = requests.get(URL, timeout=30)

with open("with_captcha.html", "w", encoding="utf-8") as f:
    f.write(response.text)

print("Saved response to with_captcha.html (status:", response.status_code, ")")

Open with_captcha.html in a browser. A normal product page means your IP and pattern are still "healthy"; a CAPTCHA / robot-check page means the protection layer is actively blocking you. Things that influence this: IP reputation (datacenter vs. residential), request frequency, cookies & JS execution, and historical behavior from your IP range.

3. Detecting CAPTCHA responses programmatically

In real scripts you don't want to open HTML manually. Use the shared detection logic from the amazon_captcha package — the same markers power the browser Playground, so behavior matches.

amazon_captcha_detect.pypython
import requests
from amazon_captcha import detect

URL = "https://www.amazon.com/dp/B096N2MV3H"

def fetch_with_captcha_detection(url: str) -> requests.Response:
    resp = requests.get(url, timeout=30)
    result = detect(resp.text, status=resp.status_code, url=str(resp.url))
    print("Status:", resp.status_code)
    if result.is_captcha:
        print(f"[WARN] CAPTCHA / bot-check page detected (risk={result.risk_level}).")
        for marker in result.matched_markers:
            print(f"  - matched: {marker!r}")
    else:
        print(f"[INFO] Response looks like a normal page (risk={result.risk_level}).")
    return resp

fetch_with_captcha_detection(URL)
🎯

This isn't bullet-proof (Amazon can change wording), but it's good enough to avoid feeding CAPTCHA HTML into your parsers and to add logging like "we're being challenged, back off or switch strategy".

4. Adding browser-like headers (what helps, what doesn't)

One of the first things people try is spoofing browser headers. It can help a bit for small-scale experiments, but it does not turn off bot detection.

with a header pool + rotationpython
import requests
from amazon_captcha.headers import BROWSER_PROFILES

URL = "https://www.amazon.com/dp/B096N2MV3H"
profile = BROWSER_PROFILES[0]  # or random_profile() / rotating_profiles()

resp = requests.get(URL, headers=profile.as_headers(), timeout=30)
print("Status:", resp.status_code)

Even with perfect headers, you still face JS fingerprinting, behavioral signals (too fast, too regular), and IP reputation. For real crawling this quickly becomes a losing battle if you handle everything yourself.

5. Why manual CAPTCHA bypass doesn't really scale

⚖️

Important distinction

Solving CAPTCHAs (sending images to a solver, using ML to break them) can violate terms of service and may be legally risky. Avoiding CAPTCHAs — by looking more like real users with IP rotation, realistic browser automation, rate limiting, and challenge handling — is usually the safer, infrastructure-heavy path.

Building all of this from scratch means maintaining a residential/mobile/ISP IP pool, browser automation that passes fingerprinting, robust retry/backoff logic, and continuous updates as anti-bot evolves. This is exactly the layer managed scraping platforms are designed to handle for you.

Advanced

Modern anti-bot & CAPTCHA landscape (2026)

Amazon and other large sites combine several layers of bot detection. It's useful to understand these even if you ultimately rely on Thordata to handle them.

  • IP reputation & ASN checks — datacenter IPs (especially from known cloud providers) are scrutinized more heavily than residential / mobile / ISP ranges.
  • TLS & HTTP fingerprinting — subtle details of your TLS handshake, HTTP/2 prioritization, and cipher choices can identify popular scraping libraries. (See examples/curl_cffi_tls_demo.py.)
  • Browser fingerprinting — headless modes, missing fonts, odd canvas results, and uniform viewports all contribute to bot scores. (See examples/playwright_stealth_demo.py.)
  • Behavioral analysis — very regular timing, no scrolling, zero interaction patterns are strong automation signals.

Modern scraping stacks combine high-quality rotating IPs, headless browser automation in "stealth" modes, human-like pacing & concurrency, and challenge handling. Thordata's Web Scraper Tools encapsulate these layers so your Python code stays simple:

Troubleshooting

FAQ

Scripts run but I only see 503 / "Service Unavailable" from Amazon

This is expected with plain requests and no proxy or browser layer. Use amazon_captcha_save.py to save the HTML and confirm you're seeing a CAPTCHA / bot-check page. Reduce request frequency, try a different network, or move to the Thordata-based examples.

The Thordata script says public_token / public_key are missing

Create a .env file in the project root with THORDATA_SCRAPER_TOKEN, THORDATA_PUBLIC_TOKEN, and THORDATA_PUBLIC_KEY. Make sure the values are correct and run the script from the same folder so load_env_file() finds .env.

Thordata scripts run, but some fields are None or missing

Different tools expose slightly different JSON schemas. Use print(data[0]) or pprint(data[0]) to inspect all available fields. The examples focus on a small, stable subset for clarity.

How do I turn JSON results into CSV?

Use the built-in export helper or the CLI:

export to CSVbash
python -m amazon_captcha export results.json products.csv --product
Does the package work without the Thordata SDK installed?

Yes. The amazon_captcha core (detect, headers, fetch, export, batch, CLI) only depends on requests. thordata_client is imported lazily and raises a clear ImportError telling you to pip install thordata-sdk only when you actually call it.
